# 军师 app · M0 快速路径（有时间压力下的最小可用交付）

> 日期：2026-07-04  
> 作者角色：产品负责人 + 技术负责人  
> 上位约束：本文件严格执行 `DECISIONS.md` 8 条拍板决策，尤其是决策 6（时间压力，九叔明天就能用）、决策 3（仅 iPhone+Mac）、决策 4（Qwen/DeepSeek 优先分级路由）。  
> 一句话核心：**先用一条 48–72 小时能落地的过渡方案让九叔"立刻用起来"建立信任，再用 2–4 周补上一个真正的原生 app。**

---

## 0. 诚实的前提（先说清楚，不粉饰）

有时间压力时最容易犯的错，是承诺一个"2 周做完完整原生 app"的时间表，然后跳票，反而毁掉九叔的早期信任。所以这份路径建立在三个诚实判断上：

1. **48–72 小时内做不出上架 App Store 的原生 app**，也做不出功能完整的 TestFlight 版。能做的是一个**过渡方案**——它有明确妥协（下面 Track A 会逐条列），但能让九叔今天/明天就开始"说真话 + 出结论"。
2. **一个能用的原生 TestFlight app 需要 2–4 周**，前提是 scope 被死死钉住（下面 Track B 的"不做清单"是关键）。
3. **过渡方案和原生 app 的数据要能延续**。九叔在 Track A 里说的话、建的 topic、生成的结论，不能在切到 Track B 时全部丢失（否则又是一次信任损耗）。因此 Track A 的数据格式从第一天起就按"能导入 Track B"来设计（Markdown + JSON，见 §5）。

## 1. 双轨策略总览

| | Track A：过渡方案 | Track B：原生 app |
|---|---|---|
| 目标 | 让九叔**立刻用起来**、建立"敢说真话"的信任 | 给九叔一个**独立、有安全感**的原生 app（决策 1）|
| 时间 | **48–72 小时**内交付到九叔手上 | **2–4 周**到 TestFlight |
| 形态 | iOS 快捷指令 + 语音 + 轻后端路由（主路径）；ChatGPT/Claude 项目式（备选）| SwiftUI 原生 app（iPhone + Mac）+ CloudKit 同步 |
| 承担的功能 | 语音→转写→确认→军师回应→当天结论落盘 | 全部 M0 必须功能（见 §3）|
| 明确妥协 | 无原生 UI；同步靠 iCloud 文件夹；无端侧加密；反昏君靠 system prompt | 妥协少，但 scope 收紧（见 §4 不做清单）|
| 谁做 | AI agents 写脚本/后端 + Moses 配置 API key + 终审人格 prompt | AI agents 写 Swift + Moses 终审 + 九叔 co-design 反馈 |

**主路径选择结论**：Track A 主路径选 **「iOS 快捷指令（Shortcuts）+ 语音输入 + 一个极轻的模型路由后端 + iCloud Drive 落盘」**。备选是 **「直接用 ChatGPU/Claude 的 Projects + 一份军师 system prompt + 手动出报告」**。

**为什么主路径不是 TestFlight 极简版**：TestFlight 极简版本质仍是原生 app，48–72 小时做不完（要过 Xcode 工程、签名、CloudKit 配置、TestFlight 审核），它其实就是 Track B 的早期形态，不该塞进"48 小时"里。**为什么不是纯 PWA**：决策 3 排除 Web 为长期形态，且 PWA 在 iOS 上语音/后台/通知体验受限，只适合桌面临时用，不适合"九叔手机随时说"的核心场景。**快捷指令**是唯一能在 48–72 小时内、在九叔 iPhone 上给出"按一下就能对军师说话"体验、且零 App Store 审核的方案。

## 2. Track A：48–72 小时过渡方案（主路径详解）

### 2.1 九叔那一侧的体验（他看到什么）

1. 手机主屏幕上有一个图标（外显名，见 NAMING.md；图标 discreet，不写"军师"）。
2. 点一下 → 弹出"说点什么"→ 他**按住说话**（用系统语音听写，中文实测够用）。
3. 松手 → 屏幕显示转写文字 → 他可以**快速改错**（人名、公司名、金额——这是 research 01 强调的确认闸门）。
4. 确认 → 几秒后军师**用文字回应**：接住他说的、给判断、必要时顶一句（反昏君常驻）、留一个追问。
5. 这次对话**自动存进 iCloud Drive** 的一个文件夹里（按 topic + 日期），Mac 上打开"文件"app 就能看到当天的结论。
6. 每天最后一次说话后，军师生成一份**当天结论快照**（标题 + 当前立场 + 新证据 + 未决问题 + 下次从哪继续），存成一个 Markdown 文件。

对九叔来说，这就是"一个能说话、会回话、会记账的军师"，他不需要知道背后是快捷指令还是原生 app。

### 2.2 技术实现（Moses / AI agents 那一侧）

```
iPhone 快捷指令
  → [听写] 系统语音转文字（中文，端侧优先）
  → [显示并允许编辑] 确认闸门
  → [POST] 发到轻后端（模型路由器）
        ↳ 后端注入：军师 system prompt（人格规范书）+ 当前 topic 的活文档 + 最近快照
        ↳ 按 §6 路由表选模型（日常 Qwen/DeepSeek）
        ↳ 返回军师回应 + 结构化字段（新判断/矛盾/追问）
  → [显示] 军师回应
  → [写文件] 追加到 iCloud Drive/军师/{topic}/{date}.md
  → [每日收尾] 生成当天快照 md
```

**轻后端**：一个极小的服务（Cloudflare Workers 或 Fly.io，TypeScript），只做三件事——注入 prompt + 上下文、按路由表调模型 API、返回结构化结果。**不存明文正文**（正文只落在九叔的 iCloud Drive）。API key 放后端环境变量，不落九叔手机。

**同步**：靠 iCloud Drive 文件夹在九叔 iPhone 和 Mac 间自动同步（Apple 原生，零开发）。这是妥协——不是端到端加密的 CloudKit 私有库，但对 48 小时过渡方案足够，且数据始终在九叔自己的 iCloud 里，不经 Moses 的服务器留存。

**上下文延续（简化版活文档）**：每个 topic 一个 `_立场.md`（当前立场书）+ 若干 `{date}.md`（每日快照）。后端每次把 `_立场.md` 全量注入 prompt（呼应 research 02 §3.5"活文档作常驻上下文，不靠模型记忆"）。这是活文档机制的最小实现。

### 2.3 Track A 的明确妥协（诚实清单）

- ❌ 没有原生 UI，是快捷指令的朴素界面（能用，但不精致）。
- ❌ 同步靠 iCloud Drive 文件，不是 CloudKit 加密字段；无端侧加密。
- ❌ 语音用系统听写，专有名词（NBA/Curry/五粮液/泸州老窖）需靠确认闸门人工改，无自定义热词。
- ❌ 矛盾时间线、思维画像、立场演变可视化**都没有**——只有"活文档 + 每日快照"两层，第三层（语义索引/画像）不做。
- ❌ 反昏君靠 system prompt 保证，没有强制的"反对意见配额"结构位校验（靠 prompt 尽力，不保证每次都有）。
- ✅ 但："语音→确认→军师回应→当天结论落盘→跨设备可见"这个**核心闭环今天就能跑**，九叔能真的开始用。

### 2.4 Track A 的 48–72 小时任务分解

| 时段 | 任务 | 谁 |
|---|---|---|
| 0–12h | 定 system prompt（人格规范书 v1 的可执行版）+ 定路由表 + 开通 Qwen/DeepSeek API | Moses 终审，AI agent 起草 |
| 12–36h | 写轻后端（prompt 注入 + 路由 + 结构化返回 + 不留存正文）| AI agent 写，Moses review |
| 36–54h | 做 iOS 快捷指令（听写→确认→调后端→显示→写 iCloud 文件）+ 每日快照脚本 | AI agent + Moses 在自己手机上调 |
| 54–66h | Moses 自测 3 个 topic（李宁 / 白酒 / 腾讯阿里续约），跑通闭环 + 改 prompt 分寸 | Moses |
| 66–72h | 装到九叔手机 + 15 分钟当面教他"按住说话/改错/看当天结论" + 交一页纸使用说明 | Moses |

## 3. M0 必须有的功能（Track B 原生 app 的验收范围）

Track B（2–4 周 TestFlight）必须覆盖以下功能，一个都不能少：

1. **语音→确认→入库**：按住说话 → 实时转写草稿（SpeechAnalyzer iOS 26+ 端侧优先）→ 停止后进入"提交前确认编辑"→ 人工改错 → 入库。入库同时生成：原文、清洗文、关键词、topic 关联（呼应 research 03 §3.3.2）。
2. **topic（话题工作台）**：可创建/选择战略项目（如"腾讯阿里续约""李宁""白酒"）。每次输入归属到一个 topic（可跨多 topic）。
3. **每日快照**：每个 topic 每天/每次会话生成一份**不可变**的当天截面结论（日期 + 标题关键词 + 当前立场 + 新证据 + 未决问题 + 下次从哪继续），可搜索、永不覆盖（research 02 §4.1 第 2 层）。
4. **活文档（当前立场书）**：每个 topic 一份始终最新的"当前立场"文档，每次对话开始整体加载进上下文（research 02 §3.5 第 1 层）。这是"结论会生长"的载体。
5. **CloudKit 同步**：iPhone ↔ Mac 同一 Apple ID 私有库同步（加密字段 + Asset），同步延迟目标 <10 秒（research 03 Alpha DoD）。
6. **反昏君开关常开**：反迎合协议 + 反对意见配额 + disruptive 创意作为结构位常驻，**不可关闭**（决策 5）。力谏强度可调，但底线不可关。
7. **可调节力谏强度**：至少 1–5 档或"自动检测主人状态 + 手动 `/max` 升档"（人格规范书交付机制细节）。
8. **模型分级路由**：日常 Qwen/DeepSeek，深度/反昏君可上最强通道（见 §6 路由表），UI 显示当前处理商与保留策略。
9. **关键词检索 + 时间线回放**：能命中最近 30 天内容，能按 topic 看历史立场串。

## 4. M0 明确不做的功能（防 scope creep）

以下功能在 M0（Track A + Track B）**明确不做**，后置到 M1.5/M2：

- ❌ **矛盾冲突图谱可视化**（立场演变时间线的图形渲染）——M0 用文字记录"转变原因+日期"即可，不做图。
- ❌ **思维画像高级提醒**（长期行为建模、短板反向提醒的自动触发）——M0 不做自动画像，Track B 最多记录数据、不主动归纳提醒。
- ❌ **PPT 深度解析**（决策附录 A #4）——M0 文件上传最多 PDF/JPG，PPT 后置。
- ❌ **端侧本地大模型完整闭环**——M0 用云模型，不做本地推理。
- ❌ **企业级审计 / 细粒度策略引擎 / DPA 流程**——单人使用，M0 不需要。
- ❌ **语义向量检索**——M0 用关键词/全文检索（SQLite FTS5）即可，向量索引后置。
- ❌ **Android / Windows / Web 原生版**（决策 3）——永不做（Web 仅 Track A 桌面临时用）。
- ❌ **多用户 / 共享 / 协作**——军师是 n=1 的私人产品。

> 判断标准：任何"让军师更聪明/更全能"但**不影响"九叔敢说真话 + 每天出结论"核心闭环**的功能，M0 一律砍掉。

## 5. 数据延续：Track A → Track B 不丢数据

Track A 的数据从第一天起按"可导入原生 app"设计：

- 每个 topic = 一个文件夹：`军师/{topic}/`
- 当前立场书：`_立场.md`（结构见 research 02 §4.1）
- 每日快照：`{YYYY-MM-DD}.md`（front-matter 里放结构化字段：title / keywords / confidence / open_questions）
- 原始输入 + 军师回应：`entries.jsonl`（每行一条 `{timestamp, topic, raw_text, edited_text, response, model}`）

Track B 上线时，写一个一次性导入器：扫描九叔 iCloud Drive 的 `军师/` 文件夹 → 映射到 CloudKit 的 `Topic / Entry / SnapshotDaily / LivingPosition` 实体（schema 见 research 03 §3.6.1）。九叔切换到原生 app 后，**历史一条不丢，直接接着用**。

## 6. 模型路由表（M0 落地版）

严格执行决策 4：大陆日常 Qwen/DeepSeek 优先；深度/反昏君可上最强；利益冲突屏蔽生效。

| 场景 | 首选模型（大陆可触达）| 通道 | 说明 |
|---|---|---|---|
| 语音纠错 / 结构化抽取（低风险高频）| DeepSeek V4 Flash 或 Qwen Flash | 国内 API 直连 | 成本最低，够用即可 |
| 日常"军师对话"| **Qwen3.7 Max 或 DeepSeek V4 Pro** | 国内 API 直连 | 主力，稳定可达，中文好 |
| 每日结论生长 / 矛盾整合 | Qwen3.7 Max（默认）| 国内 API 直连 | 大陆日常仍走国内最强 |
| **深度推理 / 反昏君 / disruptive 创意 / 重大决策**| 首选国内最强（Qwen Max 高档 / DeepSeek 高档）；**网络允许时可切国际通道**：Claude Opus / GPT-5 Pro | 双通道：国内直连 + 国际（可选）| 决策 4 双通道设计；`/max` 触发 |
| 高敏 / 利益冲突话题（如谈腾讯）| 中立通道，**屏蔽利益相关方托管**（谈腾讯不用腾讯云）| topic 级 blocklist | research 03 §3.2.3 |

**通道设计要点**：
- 默认通道 = 国内直连（Qwen/DeepSeek），保证九叔在大陆随时可用。
- 国际通道 = 可选，仅深度场景手动/自动触发；连不上时**优雅降级**回国内最强，而不是报错卡住。
- UI（Track B）显示当前会话用了哪个处理商、保留策略是什么（决策 2 的"隐私可见"）。

### 6.1 单人月费成本估算

假设：九叔单人使用，日均 15–30 次交互，其中约 80% 是日常/低风险（走 Flash/Max），约 20% 是深度场景（偶尔上最强）。

| 项 | 估算 |
|---|---|
| 日常对话（Qwen/DeepSeek，量大价低）| 约 ¥30–100 / 月 |
| 语音纠错 + 结构化抽取（Flash 档）| 约 ¥10–40 / 月 |
| 深度/反昏君（偶发上最强，国内高档为主）| 约 ¥50–200 / 月 |
| 偶尔国际通道（Opus/GPT-5 Pro，低频）| 约 ¥50–300 / 月（视频率）|
| 轻后端托管（Cloudflare/Fly 免费额度内或极低）| ¥0–50 / 月 |
| **单人合计**| **约 ¥150–700 / 月**（典型落在 ¥200–400）|

> 结论：符合决策 2"不吝啬但可控"。日常压在国内模型 → 大头成本可控；深度场景按需上最强 → 建议质量不打折。若某月国际通道用得多，费用上浮但仍在单人可接受范围。**建议 Moses 设一个月度预算上限告警（如 ¥800），超了看是否深度场景过频。**

## 7. 九叔 co-designer 参与节奏（第一周）

决策 7：九叔是信任测试官 + 需求来源，不是产品经理。第一周要拿到两类反馈：**信任反馈**（他敢不敢说真话）和**分寸反馈**（军师顶得对不对、会不会冒犯）。

### 7.1 第一周节奏

| 时间 | 要九叔做什么 | 要收集什么 |
|---|---|---|
| Day 1（交付日）| 当面 15 分钟教学；让他现场说一条真实的话（建议从"腾讯阿里续约"或最近一件烦心事开始）| 他第一次说话时**犹豫不犹豫**？哪里让他觉得不安全？|
| Day 1–3 | 让他每天至少说 3 次，覆盖 3 个 topic | 语音改错累不累？军师回应有没有"说到点上"？|
| Day 3 | Moses 打一个 10 分钟电话问反馈 | **信任测试**：问他"你敢在这里说腾讯续约的真实想法吗？还是你还在自我审查？"|
| Day 4–6 | 故意制造一次"军师顶他"的场景（Moses 可在 prompt 里稍升力谏档，或等自然发生）| **分寸测试**：军师顶他之后，他觉得被冒犯了、还是被点醒了？|
| Day 7 | 收集他主动提的优化/需求（co-designer 输出）| 他希望军师**多做什么、少做什么**？记进需求池 |

### 7.2 "敢不敢说"信任测试的收集方法

不要直接问"你信任它吗"（他会客气地说信）。改用**行为信号 + 具体场景问法**：

- **行为信号**：看他实际说的内容深度。如果他只说"今天开了个会"这种表面话，说明还不敢；如果他开始说"我其实觉得阿里那边报价是虚的，但我不敢在董事会讲"，说明树洞建立了。
- **场景问法**：给他一个具体、敏感的话题让他试（"你要不要跟军师说说你对某人续约态度的真实判断？"），然后问"说完之后，你有没有担心这话会传出去？"
- **对照问法**：问"同样这话，你会跟你的助理说吗？跟 Moses 说吗？跟军师说吗？"——如果军师排在最敢说的那档，信任成立。

## 8. 验收标准（DoD）

M0 成功的定义，不是功能清单打勾，而是**九叔的两句话**：

- ✅ **信任 DoD**：九叔（真心地，不是客气）说出类似——**"我敢在这里说腾讯续约的真实想法。"** 即他愿意把不敢对董事会/助理说的判断，说给军师听。
- ✅ **诤友 DoD**：九叔说出类似——**"它顶了我一次，但我没觉得被冒犯。"** 即反昏君真的发生了（不是一味附和），且分寸控制住了情绪（没击穿树洞）。

**辅助技术 DoD（Track B）**：
- 连续 7 天，每个活跃 topic 每天稳定产出当天快照。
- iPhone 说的话，Mac 上 10 秒内能看到。
- 关键词检索能命中最近 30 天内容。
- 反昏君结构位每次重大决策都出现（反对意见配额 ≥1 条）。

> 只要两句话中的信任 DoD 达成，即使 Track B 还没上线、九叔仍在用 Track A，M0 的**核心价值就已经交付**。功能可以慢慢补，信任窗口不能错过。

## 9. 一页纸执行顺序（给自己对齐）

1. **今天**：定 system prompt（人格规范书 v1）+ 路由表 + 开 Qwen/DeepSeek API。
2. **48–72h**：交付 Track A（快捷指令 + 轻后端 + iCloud 落盘），装到九叔手机，当面教学。
