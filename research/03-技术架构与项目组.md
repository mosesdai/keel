## 1) 问题重定义（先把真正问题说清）

- 用户表面需求：做一个“私人 AI 军师”，支持跨设备、语音优先、长期学习、强隐私。
- 实质问题：在 **“极高隐私”** 与 **“云端最强模型推理”** 之间建立可控折中，且在中国可稳定使用。
- 本报告能保证：给出可落地的架构、隐私分层策略、模型路由、项目组织、DoD 与里程碑。
- 本报告不能保证：第三方模型/云服务商政策长期不变；跨境网络质量稳定；某家模型长期保持最强（这些都需持续评测和合同约束）。

## 3) 技术架构研究（A 部分）

## 3.1 同步与存储：CloudKit vs 自建云 Hub vs 混合

### 方案 A：CloudKit + Advanced Data Protection（ADP）优先

**优点**
- Apple 生态原生多设备同步体验最好（iPhone/Mac/iPad 同 Apple ID）。
- 开启 ADP 后，多数 iCloud 类别端到端加密；第三方 app 数据中，CloudKit 加密字段与资产可被端到端保护。
- 对“树洞”心理安全感强。

**缺点**
- CloudKit 公共库不加密；私有/共享库才有账户级保护。
- CloudKit 加密字段不能建索引，且已有明文字段不能直接改成加密字段，影响服务端检索设计。
- 若内容端到端加密，服务端无法直接做明文 AI 推理（必须客户端解密后再发起调用，或保留一份可服务端处理的脱敏副本）。

### 方案 B：自建云 Hub（后端中心化）

**优点**
- 服务端统一做 AI 编排、日志、评测、A/B、任务队列、文件解析，工程效率高。
- 模型密钥管理与成本控制更容易。

**缺点**
- 隐私信任门槛高，用户会担心“真心话是否被留存/复用”。
- 一旦日志、可观测、排障链路没做好，隐私事故风险大。

### 方案 C：混合（推荐）

**核心思想：把“同步”和“算力”解耦**
- 同步主干：CloudKit 私有库（加密字段 + 资产）。
- AI 推理：**客户端优先发起**（高敏），轻后端只做策略与短期令牌，不碰原始内容；低敏任务可走后端统一编排。
- 搜索：端侧全文索引 + 语义索引为主，云端只放必要的检索投影（标题、关键词、时间、topic id）。

**结论**
- 该方案最符合“多设备同步 + 不敢说变敢说 + 仍能用最强模型”的组合目标。

## 3.2 隐私与算力核心矛盾：供应商政策与推荐分层

## 3.2.1 已核实的关键政策（摘要）

### DeepSeek 官方
- 隐私政策写明：会为“改进服务/训练模型”使用输入输出；可通过关闭“数据用于优化体验”退出。
- 数据处理与存储写明位于中国（PRC）。
- 保留周期为“为实现服务目的所必需”，未给固定最短天数上限。

### 国内云托管 DeepSeek（代表性）
- **阿里云百炼**
  - 官方隐私说明写明“不会将您的数据用于模型训练”。
  - 服务协议写明“未获明确授权不使用对话数据训练模型”，并承诺仅在必要最短时间保留对话数据。
  - 同时协议也明确：若调用三方模型，第三方协议可能单独适用（需逐项确认）。
- **腾讯云（大模型服务条款）**
  - 条款写明数据处理地点为中国境内、仅在必要期间保留、到期返还或删除。
  - 未在本次检索到的条款中看到与阿里同等强度的“默认不用于训练”明确句式，建议按【待核实】处理并走合同条款补充。
- **火山方舟**
  - 社区文章宣称“提示词会话数据零留存、安全沙箱”；该表述来自官方社区文章，不等于法律合同文本，建议【待核实】并以企业合同为准。

### 国际选项
- **OpenAI API**
  - API 文档写明训练项为 No，默认 abuse log 最长 30 天。
  - Zero Data Retention（ZDR）/Modified Abuse Monitoring 需审批，不是默认开关。
- **Anthropic API**
  - 商业 API 默认后端 30 天删除输入输出（有例外：法律、违规、特定功能）。
  - ZDR 需审批，且仅对合格 API/企业产品生效。
  - 部分 Covered Models 有 30 天保留要求（即使总体是 ZDR 组织）。
- **Google Gemini API**
  - Paid Services 下：不使用 prompts/responses 改进产品。
  - 但会为安全合规进行有限日志；可申请项目级 ZDR。
  - Grounding with Google Search/Maps 明确有 30 天存储（不可关闭）。
- **Azure OpenAI（Microsoft Foundry 中 Azure 托管模型）**
  - 文档明确：提示词/输出不提供给 OpenAI，不用于训练基础模型（未经客户授权）。
  - 提供地理与数据驻留控制，企业合规路径成熟。
- **AWS Bedrock**
  - FAQ/安全页写明：内容不用于改进基座模型，默认不与模型提供方共享。
  - 但 2026 新 retention 模式下，部分模型（如 Claude Fable 5 / Mythos 5）需要 provider_data_share，且可达 30 天安全留存；需按模型逐条配置。

## 3.2.2 “隐私-算力分层”推荐（落地版）

定义四级数据敏感度（L0-L3）：

- **L3（最高敏）**：对关键人物评价、谈判底牌、未公开商业动作。
  - 默认：端侧处理 + 本地摘要 + 必要时先脱敏再上云。
  - 云侧仅允许：已签 ZDR/强合同约束通道。
- **L2（中高敏）**：战略判断、内部组织问题。
  - 可用高质量云模型，但必须经过最小化与可逆映射（实体代号化）。
- **L1（中低敏）**：通用分析、公开事实整理。
  - 可走性价比模型（DeepSeek/Qwen/Flash 系列）。
- **L0（公开）**：公开资料、新闻摘录。
  - 成本优先路由。

## 3.2.3 “利益冲突路由”必须显式配置

针对“讨论腾讯/阿里谈判策略是否可用其云托管模型”的顾虑，建议在系统策略层加入：

- topic 级 `conflict_of_interest_provider_blocklist`（如 topic=腾讯续约时，禁用腾讯托管通道）。
- 支持用户一键切换“中立提供商模式”（例如 Azure/OpenAI/Anthropic 或本地模型优先）。
- 在 UI 显示当前会话的“处理商与保留策略”，让用户实时可见，不做黑箱。

## 3.3 语音输入方案（iOS）

## 3.3.1 三种路径对比

1. **Apple SpeechAnalyzer（iOS 26+）**
   - 官方 WWDC 说明：新模型更快、更灵活，适合长语音和远场音频；端侧运行。
   - 优势：隐私、低延迟、系统级集成。
   - 风险：特定行业词与中英夹杂准确率需要实测调优。

2. **SFSpeechRecognizer（兼容旧系统）**
   - `requiresOnDeviceRecognition=true` 可强制不上传网络，但 Apple 明示端侧精度可能低于在线路径。
   - 适合：老系统兜底。

3. **whisper.cpp（端侧开源）**
   - 优势：可私有化、可控。
   - 风险：在中文场景，公开社区基准显示与中文专用 ASR 有明显差距；移动端大模型内存/时延压力高。

4. **云端 ASR（兜底）**
   - 优势：专有名词可通过热词/词表提升（需选定供应商后压测）。
   - 风险：音频上云，隐私权衡明显。

## 3.3.2 推荐交互流（必须实现）

`按住说话 -> 实时转写(草稿) -> 停止后进入“提交前确认编辑” -> 用户修正 -> 入库`

入库时同时生成：
- 原文（可加密）
- 清洗文（用于检索投影）
- 关键词（可搜索）
- topic 关联

## 3.4 模型选型（2026）与路由建议

## 3.4.1 可用模型池（按“强推理+商业创造力+可调用”）

- **最高能力层**：Claude Fable 5 / Opus 4.8、GPT-5.5 / GPT-5 Pro（按可得性与合同）
- **高性价比层**：DeepSeek V4 Pro / V4 Flash、Qwen3.7 Max / Plus、Gemini 3.1 Pro/Flash 系
- **轻量任务层**：各家 mini/flash/nano

参考点（已核实）：
- OpenAI `gpt-5`：400k context，$1.25/$10（输入/输出，1M token）。
- OpenAI `gpt-5-pro`：400k context，$15/$120。
- Anthropic：Sonnet/Opus/Fable 体系，1M context（特定模型），Sonnet 5 在 2026-08 前有促销价。
- DeepSeek V4 Flash/Pro：1M context，价格显著低。
- Qwen Cloud `qwen3.7-max`：价格 $2.5/$7.5，长上下文分层计费。
- Gemini 3.1 Pro：官方文档给出 1M / 64k（in/out）与 >200k 阶梯计费。

## 3.4.2 推荐路由（务实版）

- **语音纠错/结构化抽取（低风险）**：DeepSeek V4 Flash 或 Qwen Flash（低成本）。
- **日常“军师对话”**：Claude Sonnet 5 / GPT-5（平衡质量与成本）。
- **“每日结论生长”+矛盾整合+重大决策讨论**：Claude Opus 4.8 或 GPT-5 Pro（高推理档）。
- **高敏会话**：优先本地预处理 + 中立高保障通道（ZDR/合同约束），默认禁用利益相关方托管。

> 说明：顶级模型排名会波动，建议每周自动跑一次内部评测（见 5.2 DoD）做动态路由。

## 3.5 文件处理：PDF / PPT / JPG

推荐两段式：

1. **解析层**
   - PDF：先提文本层；无文本层再 OCR。
   - PPT/PPTX：提取页标题、要点、备注；保留页级锚点。
   - JPG/截图：OCR + 版面结构（表格/图表区域）识别。

2. **理解层**
   - 低成本文本理解：先做 chunk + 摘要 + 关键词。
   - 高价值页面（图表、组织图、复杂表格）：再调用视觉模型补充语义。

入库建议：
- 原文件放 `Asset`（CloudKit 资产默认加密；私有库）。
- 派生文本与页级摘要分开存，便于检索和复算。

## 3.6 “生长性”数据模型设计

满足需求：topic -> entries（时间流）-> daily snapshots -> living document（当前立场）-> 可回看矛盾历史。

## 3.6.1 核心实体（建议）

- `Topic`
  - `topic_id`, `name`, `status`, `owner`, `sensitivity_level`
- `Entry`
  - `entry_id`, `topic_id`, `timestamp`, `input_mode(voice/text/file)`, `raw_text`, `edited_text`, `source_refs`
- `SnapshotDaily`
  - `snapshot_id`, `topic_id`, `date`, `title`, `summary`, `keywords[]`, `confidence`, `open_questions[]`
- `LivingPosition`
  - `topic_id`, `current_position`, `assumptions[]`, `last_updated`
- `ContradictionEdge`
  - `from_entry_id`, `to_entry_id`, `conflict_type`, `explain`
- `CognitiveProfile`
  - `user_id`, `thinking_patterns`, `blind_spots`, `reminder_rules`, `evidence`

## 3.6.2 检索设计

- **全文检索**：端侧 SQLite FTS5（主检索）；跨设备同步仅同步必要索引投影（标题/关键词/时间/topic）。
- **语义检索**：端侧 embedding（可小模型）+ ANN 索引；高敏默认不上传向量。
- **注意 CloudKit 限制**：加密字段不可索引，因此“检索投影”与“敏感正文”要分离存储。

## 3.6.3 矛盾处理机制（产品关键）

- 不是“判对错”，而是“时间线并存 + 条件触发切换”。
- 每条结论必须带：生效条件、证据来源、反证状态。
- Daily Snapshot 只做“当日截面”，不覆盖历史立场。

## 3.7 推荐总体架构（文字图）

`[iPhone/Mac/iPad SwiftUI App]`
-> 语音采集 & 本地转写（SpeechAnalyzer/SFSpeech）  
-> 本地敏感检测与脱敏（可选）  
-> 本地存储（SwiftData/SQLite FTS/向量索引）  
-> CloudKit 私有同步（加密字段 + Asset）  
-> 模型路由器（轻后端：策略、配额、短期令牌、审计）  
-> 多模型 API（主力/备选/成本层）  
-> 回写 Snapshot + LivingPosition + Profile

## 技术栈建议

- 客户端：SwiftUI（iOS + macOS），Combine/Swift Concurrency。
- 同步：CloudKit 私有库 + 共享库（必要时）。
- 本地数据：SwiftData + SQLite FTS5 + 向量库（如 sqlite-vss 或等价方案）。
- 轻后端：TypeScript（Cloudflare Workers/Fly.io）或 Go（低运维）；只做编排与策略，不长期存明文。
- 可观测：最小化日志（脱敏后），默认关闭 prompt 全量日志。

## 5) 作业标准、质量与检验（DoD）

## 5.1 阶段 DoD（完成定义）

### 架构冻结 DoD
- 数据分级（L0-L3）文档化并在代码中可执行。
- 至少 2 条模型路由策略可切换（高敏/常规）。
- 删除与导出机制可演示。

### Alpha DoD
- 语音 -> 转写 -> 人工修正 -> 入库链路跑通。
- iPhone 与 Mac 同步延迟可接受（目标 <10 秒，网络正常）。
- 日截面快照自动生成准确率通过人工抽检。

### MVP DoD
- 每个 topic 能形成连续时间线与每日快照。
- 关键词检索可命中最近 30 天内容。
- “矛盾观点并存”可查看，不被系统粗暴覆盖。

## 5.2 军师回答质量评测（重点）

建立 **反迎合测试集**（必须）：
- 故意输入前后矛盾、明显错误判断、情绪化偏见。
- 看模型是否能指出冲突、提出反证、给条件化结论。

建议指标：
- `Challenge Rate`（该质疑时是否质疑）>= 70%
- `Contradiction Catch Rate`（矛盾识别）>= 85%
- `Evidence Citation Rate`（给出处/给待核实标记）>= 90%
- `Sycophancy Rate`（无脑附和）持续下降并有周报。

## 5.3 语音准确率标准（建议）

- 安静办公场景中文口述：CER <= 8%（提交前纠错后）。
- 中英夹杂品牌词（NBA、Curry、五粮液）关键词召回 >= 95%（以自定义词表/后处理校正规则统计）。
- 实时字幕端到端延迟：P50 < 1.5s，P95 < 3s。

> 以上阈值建议先用九叔真实语料做 2 周校准后定板。

## 5.4 隐私检验清单（上线门禁）

- [ ] 高敏数据默认不进入低保障通道。  
- [ ] 供应商保留策略在 UI 可见（当前通道、保留天数/模式）。  
- [ ] 关闭/删除账号后，云端与本地删除流程可验证。  
- [ ] 日志、监控、错误上报不含原始高敏内容。  
- [ ] API key 不落客户端明文；支持轮换。  
- [ ] 供应商条款变更有告警与重新评审流程。  

## 5.5 “生长性机制”验收

- 连续 14 天测试中，daily snapshot 每天稳定产出。
- 同一 topic 可看到“历史立场串”，且系统不会强制单一裁决。
- 用户可一键查看“我为什么现在这样判断”（证据链可追溯）。

## 6) 研发过程学习与经验沉淀机制

建议直接继承你现有 SESSION_LOG 工作法，升级为四层沉淀：

1. **Decision Log（决策日志）**
   - 每个关键决策记录：背景、选项、取舍、风险、复盘时间。
2. **Experiment Ledger（实验台账）**
   - 每次模型路由/Prompt 调整记录：输入集、结果、指标变化。
3. **Pattern Library（可复用套路库）**
   - 把有效提问框架、反迎合提示、冲突处理模板沉淀成模板。
4. **Release Review（版本复盘）**
   - 每个里程碑后沉淀“做对了什么、踩了什么坑、下版禁做清单”。
